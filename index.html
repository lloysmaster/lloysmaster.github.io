Arquitectura del Firmware (RP2040 Dual-Core)

El firmware aprovecha la arquitectura de doble núcleo para separar las tareas de tiempo real crítico de las comunicaciones.

Core 0: Control Crítico

  • Loop de Control: Ejecución del PID y Mezclador (Mixer).
  • Sensores: Lectura del MPU6500 vía SPI a alta velocidad.
  • Salida Motores: Protocolo DShot300 implementado mediante PIO.
  • Filtros: Aplicación de filtros EMA para reducción de ruido en gyro.

Core 1: Gestión y Telemetría

  • Radio: Recepción de señal PPM mediante PIO y sincronización con Core 0.
  • Consola: Manejo de comandos seriales para ajuste de PIDs en caliente.
  • Telemetría: Despacho de datos por usbSerial (~25Hz).

Objetivos de Aprendizaje y Optimización

  • Implementación de PID en punto fijo (q16.16).
  • Incremento del lazo de control a 8kHz.
  • Uso de multiplicaciones rápidas en la ROM del RP2040.

Análisis de Propulsión (MT2204)

Según las pruebas de rendimiento, estos son los parámetros de operación para el cálculo del Mixer y PID:

  • Eficiencia: Máxima con hélices 5045 (6.53 g/W a baja potencia).
  • Empuje Máximo: 430g por motor con hélices 6x3 @ 13.1A.
  • Seguridad Térmica: Los motores alcanzan 50°C en ráfagas de máxima potencia; se recomienda monitoreo.

Requisitos del Sistema de Energía

  • Entrada: Soporte hasta 26V (Batería 6S) para futura escalabilidad.
  • Salida: 5.0V - 5.2V estables para alimentar el RP2040 y el receptor Futaba.
  • Filtrado: Frecuencia de conmutación > 500 kHz para evitar ruido en las lecturas del giroscopio.
